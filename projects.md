## Simplicity Rules

- Do not display projects as decorative achievement pills, badges, or chip rows.
- Do not repeat the same project summary on the homepage, projects index, and detail page without tightening it for each context.
- Use plain filters, direct links, and clear project titles.
- Use cards only for repeated project records; avoid nested cards or decorative wrappers.
- Keep project pages evidence-led: overview, what was reported, location, date, sources, and related projects.

## Publishing Rules

- Use "reported" where the only evidence is media reporting.
- Use direct dates only where the source gives a date.
- Do not publish project costs, beneficiary totals, or medical claims unless the client provides documentation.
- Use client-owned or licensed photographs only.
- Add source links to the CMS for editor review, even if the live site does not display every link.
- Avoid claims like "largest", "first", "only", "best", or "most impactful" unless proven by independent records.
